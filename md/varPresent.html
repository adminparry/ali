变量提升机制

js变量提升机制

这是一道阿里的前端面试题


var a = 10;
(function () {
    console.log(a)
    a = 5
    console.log(window.a)
    var a = 20;
    console.log(a)
})()

依次输出：undefined -> 10 -> 20

又是一道陷阱题 关键的点就是在这个var a = 20;

如果是从上往下看那么第一个a访问的是全局的10 然后window.a 是改变后的5 这一切都很正常 但是就是这个var a = 20 又定义了个重名的变量 没有最垃圾只有更垃圾 如果这个地方定义的是别的变量也没有问题 所以改变了整个的结果

var a;
a = 10;
(function(){
    var a;
    console.log(a) //undefined
    a = 5;
    console.log(window.a) //10
    a = 20;
    console.log(a) // 20
})()
